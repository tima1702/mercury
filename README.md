# Заявка на займ — тестовое задание

SPA из трёх шагов: личные данные → адрес и место работы → параметры займа.
Данные собираются в общий стейт, на третьем шаге уходят в `dummyjson`,
после ответа показывается модалка с подтверждением.

## Стек

- **Vite + React 19 + TypeScript**
- **react-router-dom** — переключение между шагами (требование ТЗ).
- **react-hook-form + zod + @hookform/resolvers** — валидация и работа с
  формами без ручных `useState` на каждое поле, с типизированной схемой.
- **react-international-phone** — поле телефона с селектором страны и
  автоматической маской под формат выбранной страны. ТЗ требовало маску
  `0XXX XXX XXX` (украинский формат) — её мы получаем по умолчанию,
  т.к. дефолтная страна `ua`, но пользователь может сменить страну, и маска
  переключится. Лёгкая (без libphonenumber-js), флаги встроены.
- **bootstrap** (только CSS) — базовая стилизация форм и модалки.

Глобального стора нет: данных немного, обходимся `useReducer` в Context.

## Запуск

```bash
npm install
npm run dev
```

Сборка прод-версии:

```bash
npm run build
npm run preview
```

## Что проверить

1. На первом шаге — обязательные поля и формат телефона `0XXX XXX XXX`.
2. На втором — селект «Место работы» подгружается с
   `dummyjson.com/products/category-list`. Промис кешируется
   (см. `src/api/products.ts`), повторных запросов нет.
3. Кнопка «Назад» на шаге 2 и 3 сохраняет введённые данные.
4. Прямой переход на `/step2` или `/step3` без заполнения предыдущих
   шагов редиректит на `/`.
5. На шаге 3 — два ползунка ($200…$1000 шаг 100, 10…30 дней шаг 1).
   После «Подать заявку» уходит `POST /products/add`,
   затем модалка вида:
   `Поздравляем, <Фамилия> <Имя>. Вам одобрена $<сумма> на <срок> дней.`
